Урок 02 — лайв Q&A

Масштабирование без оверинженеринга

Еженедельный разбор вопросов: когда один HQ превращается в сеть проектов, где хранить правила, как переиспользовать пайплайны через скиллы и MCP — и почему не стоит строить систему для нерешённой задачи.

№ выпуска
II · 23·04·26
формат
live Q&A
§ I.

Термины

Слова, которыми Серёжа режет урок на слои.
01

Стратегия / исполнение

Метафора офиса: сверху — директор, снизу — экран и мышь. Задача штаба — как можно реже спускаться в исполнение, оставаясь на уровне стратегии.

02

Оркестратор — штаб

Папка-координатор с правилами, картой проектов и ссылками наружу. Сам задачи не делает — делегирует в проекты-исполнители.

03

Скилл

Упакованная экспертиза: папка с `skill.md` и файлами. Говоришь «сделай скилл» прямо в чате, когда результат понравился — и он пакует рецепт.

04

MCP-сервер

Обёртка над папкой-проектом: частые запросы становятся инструментами агента. Переход от папки к MCP — сигнал, что агент теряется в сырых правилах.

05

Hermes

Агент-продукт, который на каждое удачное действие сам создаёт скилл и в следующий раз вызывает его. Автоматизация ручного паттерна «упаковываю в скилл».

06

Superpowers

Набор мета-скиллов — не про домен, а про работу с агентом: Writing Skills, Systematic Debugging. Применяются к любой задаче, а не к конкретной нише.

07

Evals / самопроверка

Критерии, по которым агент сам оценивает свой результат. Простая версия — 10-балльная шкала и чек-лист: если меньше 6 — переделывай или правь скилл.

08

Human Gate

Точка обязательного подтверждения человеком. Прописывается в правилах приватных/рабочих штабов, где автономия агента опасна.

09

Headless-режим

Запуск агента без интерфейса, через CLI. Один агент вызывает другого и складывает результат в общий файл — вместо ручной пересадки контекста.

10

GitHub Projects

Бесплатный реестр задач, которым рулит агент. Оркестратор создаёт карточку, исполнитель забирает по ссылке — карта проектов без отдельной CRM.

§ II.

Методы

Принципы, с которых Серёжа начинает любой ответ на «как масштабировать».
01

Не строить систему для нерешённой задачи

Вначале — как получится, в одной папке. Система строится только когда упёрся в реальное ограничение. Иначе — прокрастинация под видом инженерии: друг Серёжи 8 месяцев собирал контент-завод вместо первого выпуска.

02

Сначала в проекте, потом — в общий слой

Любой переиспользуемый инструмент (генерация, парсинг, презентации) сначала делается внутри конкретного проекта. Только когда научился — выносится в скилл, MCP или отдельный сервис.

03

Правила на двух этажах: штаб + проект

В штабе — глобальные принципы и карта проектов. В папке проекта — домен-специфичные правила (код, БД, публикация, деньги). Один-два проекта — можно всё вместе.

04

Надзиратель-агент по расписанию

Отдельная функция агента на кроне: читает правила и данные, выкидывает устаревшее, синхронизирует источники (например, Telegram → CRM). Правила сами не чистятся.

05

Самопроверка через критерии, а не общие принципы

Общие «не ленись» в AGENTS.md не работают — модель и так знает. Работает чек-лист под задачу: штампы, запрещённые слова, 10-балльная шкала. Меньше 6 — начать заново или обновить скилл.

06

Writing Skills + TDD для упаковки экспертизы

Скилл `Superpowers:Writing Skills` сначала роняет задачу (test-driven), ловит ошибки, спрашивает фидбэк и только потом пакует рецепт. Серёжа делает все свои скиллы через него.

07

Два штаба: личный + рабочий с Human Gate

Приватка и рабочее — разные штабы со ссылкой друг на друга. В приватном прописано: «нельзя автономно, всегда спрашивать подтверждение». Работает только на чистом контексте.

08

Claude — оркестратор, Codex — исполнитель для систем

Claude держит стратегию и креатив (тексты). Codex сильнее в анализе/бэкенде/БД/связях. Простое пишешь планом в Codex; сложное — Claude делегирует Codex'у через скилл оркестрации.

09

Headless-оркестрация вместо «связующего звена»

Ловишь себя на роли курьера между сессиями — формализуй действия (вход → выход) и замени себя headless-вызовом: агент A пишет в общий файл, агент B читает и отвечает.

§ III.

Инструменты

Связка Серёжи для двухэтажной работы: стратегия наверху, исполнение внизу.
Claude Code 01

Оркестратор в терминале. Opus — стратегия и креатив, Sonnet — поиск/research. Умеет вызывать Codex в headless-режиме.

Codex 02

Исполнитель для систем с множеством связей: бэкенд, аналитика, БД. С креативными текстами справляется плохо — туда лучше Claude.

Superpowers 03

Библиотека мета-скиллов. Ключевые — Writing Skills (TDD-создание скиллов) и Systematic Debugging (распутывание зависимостей).

Hermes 04

Агент-продукт, который сам упаковывает удачные действия в скиллы. Следующее поколение ручного «давай сделаем из этого скилл».

GitHub Projects 05

Бесплатный реестр задач. Оркестратор заводит карточку, исполнитель забирает по ссылке — карта проектов без отдельной CRM.

MCP-серверы 06

Обёртки над внешними сервисами: Telegram через Telethon, Chrome DevTools, Exa для скрапа. Частые запросы превращаются в инструменты.

my.telegram.org 07

Обязательная регистрация аккаунта разработчика до любой автоматизации. Иначе Telegram помечает как бота и удаляет рабочий аккаунт.

PaperClip 08

Референсный фреймворк оркестрации агентов. Любопытно попробовать, но токены утекают лавиной — «на танке за цветами».

§ IV.

Цели & домашка

Слева — что должно остаться в голове. Справа — живой статус ДЗ в твоём HQ.

Цели урока5 пунктов

  • Сигнал перехода папка → скилл / MCP
    Агент теряется в сырых правилах — пора паковать.
  • Разделение стратегии и исполнения
    Работать сверху, в исполнение спускаться как можно реже.
  • Writing Skills в бою
    Упаковать один свой удачный пайплайн через `Superpowers:Writing Skills`.
  • Личный и рабочий штабы с Human Gate
    Прописать, где агент не имеет права действовать автономно.
  • Связка Claude + Codex
    Клод — оркестратор, Codex — исполнитель анализа/системы. Проверить на одной задаче.

Домашнее задание0 из 5 закрыто

  • Зарегистрировать my.telegram.org
    До любых экспериментов с Telethon — api_id и api_hash, иначе бан рабочего аккаунта.
  • Упаковать один пайплайн в скилл
    Взять повторяющийся сценарий, пройти TDD-цикл через Writing Skills: падение → фикс → упаковка.
  • Ввести Human Gate в приватные правила
    Сверху в AGENTS.md штаба с деньгами/партнёрствами: «работать не автономно, спрашивать подтверждение».
  • Формализовать роль «связующего звена»
    Зафиксировать: получаю X от A → отдаю B. Один кусок заменить headless-вызовом.
  • Подключить надзирателя-агента
    Крон или скилл: раз в день читает правила и карточки, синхронизирует Telegram → CRM, помечает устаревшее.
