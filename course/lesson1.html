HQ — первый шаг к агент-оркестрации

Вводный практический урок: создаём папку-штаб (HQ) с единым файлом правил для агента, подключаем транскрипт и собираем HTML-дашборд — скелет персональной агент-системы.

§ I.

Термины

Словарь, без которого остальной урок не соберётся в целое.
01

HQ — штаб

Главная папка, в которой лежит весь контекст про пользователя и ссылки на все проекты. Точка старта для агента и объединяющий узел сетки.

02

AGENTS.md

Открытый формат файла правил для агентов (инициатива OpenAI). Агент читает его перед тем, как начать думать — кто он, как работает, какова структура папки.

03

CLAUDE.md

Аналогичный файл-точка входа для Claude. Обычно — зеркало со ссылкой на AGENTS.md, чтобы правила были универсальны для любого агента.

04

Markdown (.md)

Формат файлов — золотая середина между кодом и текстом. Удобно читать и людям, и агентам; на нём строится вся файловая база знаний HQ.

05

Obsidian

Бесплатная программа-интерфейс для просмотра markdown-файлов. Визуальная надстройка над той же папкой HQ, с которой работает агент.

06

Dashboard (HTML)

Один HTML-файл в HQ, показывающий текущие задачи и связанные проекты. Агент по правилу автоматически обновляет его при каждом изменении задач.

07

База знаний

Отдельный файл в HQ, куда агент записывает ключевые понятия и факты: что такое markdown, AGENTS.md, факты про людей и проекты из CRM.

08

Transcript

Текстовая расшифровка урока или разговора (Серёжа собирает через Granola). Кладётся в папку курса как сырьё для извлечения терминов, методов и задач.

09

Prompt

Текстовый запрос к агенту. Серёжа подчёркивает: общаться нужно как с человеком и вкладывать время в передачу намерения — чем подробнее, тем лучше результат.

10

Composer (Cursor)

Быстрый агент-режим в Cursor. На уроке — дефолтный инструмент, потому что очень быстро рисует HTML-дашборды и презентации.

§ II.

Методы

Практики, которые Серёжа повторяет из урока в урок.
01

Начинать с простого — один файл

Любая система начинается с одного файла правил; не строить сложные системы сразу. Усложнять только когда упрёшься в реальные ограничения.

02

Папка без кириллицы в пути

HQ и все вложенные папки/файлы создавать только латиницей; русские символы в пути могут ломать команды агента. Системные папки типа Documents — ок.

03

Локальная работа, сервер потом

Разворачивать HQ на локальном компьютере, где работаешь ежедневно. Сервер — отдельный слой friction; перенос будет позже, отдельным занятием.

04

Отдельные репозитории, а не один гигант

Каждый проект — отдельная папка/репозиторий рядом с HQ. HQ играет роль архивариуса со ссылками, а не склада всего кода в одной куче.

05

Улучшение правил через research-цикл

После первого черновика AGENTS.md отправить его в ChatGPT/Claude с поиском и промптом про best practices ведущих лабораторий; результат заменяет исходник.

06

Универсальная точка входа: AGENTS.md + CLAUDE.md

Держать оба файла; CLAUDE.md — зеркало со ссылкой на AGENTS.md, чтобы любой агент (Cursor, Claude Code, Codex) работал с одним источником правды.

07

Dashboard обновляется автоматически

В правилах агента прописать: при любом изменении задач править dashboard.html. Интерфейс живёт рядом с данными и всегда синхронен без отдельного приложения.

08

Separation: правила отдельно от данных

Один файл — инструкции и маршрутизация, остальное (задачи, стратегия, база знаний, транскрипты) — отдельные файлы. Раздулось до сотен строк — вынеси.

09

Итерация с агентом как с человеком

Перед выполнением просить агента задать 2 уточняющих вопроса с 3 вариантами ответа. Чем больше вложено в передачу намерения, тем лучше результат.

§ III.

Инструменты

Фреймворки, модели и приложения, которые Серёжа упоминает в уроке.
Claude Code 01

Основной агент Серёжи в терминале. Opus — для сложных задач, Sonnet — для поиска и research.

Cursor · Composer 02

IDE-агент для тех, кому некомфортен терминал. Composer быстро рисует HTML-дашборды и презентации.

Codex 03

Альтернативный агент; узнаваем по bento-стилю карточек. Используется наравне с Claude Code.

ChatGPT / OpenAI 04

Внешний чат с поиском, куда уходит черновой AGENTS.md для улучшения по best practices.

Granola 05

Бесплатное приложение для транскрипции встреч. Выдаёт текст, с которым можно сразу общаться; источник транскриптов для HQ.

Obsidian 06

Просмотрщик markdown-файлов HQ. Интерфейс поверх той же папки, с которой работает агент.

Zoom · Telegram 07

Каналы потока: Zoom — живые лайвы и демо; Telegram — обмен файлами и скриншотами результатов практики.

§ IV.

Цели & домашка

Слева — что должно быть после урока. Справа — живой статус ДЗ в твоём HQ.

Цели урока5 пунктов

  • Локальная папка HQ
    Без кириллицы в пути, открывается своим агентом.
  • Единый файл правил
    AGENTS.md / CLAUDE.md в HQ — самодостаточный, читается с нуля.
  • Research-цикл правил
    Прогнать черновик через ChatGPT/Claude с поиском и заменить исходник.
  • Папка курса рядом с HQ
    Personal Corp подключена, транскрипт 1-го урока сохранён.
  • Минимальный HTML-дашборд
    Обновляется агентом при изменении задач.

    Публикуется на GitHub Pages, обновляется правилом §9.
  • Добавить реальный проект в HQ
    Карточки проектов есть, но поле «Путь» в `AGENTS.md` всё ещё `_(уточнить)_`.
  • Учебные материалы из транскрипта
    Этот самый файл — выжимок 1-го урока из `transcripts/lesson1.txt`.
  • Поделиться с потоком
    Отправить скриншот HQ + dashboard в Telegram/Zoom курса.
Из уст Серёжи
Всегда надо начинать с простого.
Есть способность LLM усложнять простые вещи.
Не нужно строить сложные системы — начинается с одного файла любая система.
Важно упереться в ограничения — пока сам не поймёшь, оно как не получится.
Самый классный путь что-то выучить — начать с одного кирпича, и потом добавить ещё один, ещё, ещё.
