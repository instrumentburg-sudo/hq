# Недельный план W21 — разворот сервиса

Период: **2026-05-18 — 2026-05-24**.  
Статус: черновик по формату `weekly-planning` / `ris-prioritize`.  
Источник: `priorities/2026-05-15-service-turnaround-prioritization.md`, GitHub Project «Антон — Всё», live-снимок сервиса 2026-05-15.

## Контекст недели

Главная задача W21 — перевести стратегию сервиса из «куда расти» в «что уже в понедельник двигает деньги и разгружает операционку».

Ограничение по Антону: рабочее окно нестабильное, ориентир **4–5 часов в день**, поэтому план делится на:

- решения Антона, где нужен владелец бизнеса;
- работу агента/команды, которую можно подготовить без постоянного участия Антона;
- стоп-лист, чтобы не открыть новый фронт раньше кассовой дисциплины.

## Eisenhower split

### Важное + срочное — Антон делает сегодня/в понедельник

- Утвердить правило предоплаты запчастей для физиков.
- Утвердить минимальный чек и платную диагностику.
- Дать стоп на новые исключения по постоплате без лимита.
- Выбрать, кто делает первые 30 касаний по хвостам.

### Важное + не срочное — запланировать на неделю

- Карта 44/223-ФЗ контрактов.
- Роль координатора/диспетчера.
- WIP-лимиты в ежедневном контуре.
- Отдельный P&L для проката перед любым новым пунктом.

### Срочное + делегируемое — агент/команда

- Список хвостов и сегментация.
- Тексты сообщений клиентам по деталям, выдаче, хранению.
- Черновик вакансии координатора.
- Шаблон карты контракта.

### Не срочное / не брать

- Новый город.
- Полноценный новый пункт проката.
- Найм мастера первым.
- Большой B2B/SMB-продукт без кассовой дисциплины внутри сервиса.

## Стоп-условия недели

- Если ready/stale не разобраны — не обсуждать новую точку проката как ближайший шаг.
- Если правило предоплаты деталей не внедрено — не расширять поток ремонта.
- Если карта 44/223-ФЗ не собрана — не считать тендеры «ростом», считать их риском дебиторки.
- Если WIP растёт — останавливать новые входы в низком чеке.
