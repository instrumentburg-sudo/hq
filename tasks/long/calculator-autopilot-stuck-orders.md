# Автопилот застрявших заказов (дашборд → SMS → мотивация)

## Контекст

Много заказов не добиваются до денег — застревают на админских статусах ожидания: клиент не ответил, не дозвонились, ждём запчасти, ждём согласования сметы. Это НЕ про простой мастеров (см. отдельную задачу `calculator-master-interface`), это про «хвост» заказов, который никто не дожимает.

Рамка автопилота от 2026-04-09: `видимость → агенты по одному → мотивация последним`, 5 мёртвых статусов, цель «не висит >10 дней», без Telegram (SMS + landing), Андрей не переучивается. Память: `memory/project_workflow_autopilot_design.md`, `memory/feedback_service_two_fronts.md`.

## Цель

Долю заказов в admin-ожидающих статусах возрастом >10 дней снизить с текущей (надо замерить в Шаге 0) до <10%. Метрика и цифра «до» появляются вместе с Шагом 0.

## План

- [ ] **Шаг 0 — дашборд мёртвых статусов.** Convex query + страница в calculator: список заказов по каждому из 5 мёртвых admin-статусов с колонками «возраст в днях», «ответственный», «последнее действие». Без автоматизации, только видимость. Результат — базовая метрика «сколько висит и где».
- [ ] **Шаг 1 — SMS-согласование сметы.** Landing со сметой по токенизированной ссылке, три кнопки: согласую / отклоняю / перезвоните. SMS с коротким URL отправляется админом из calculator одним кликом. Интеграция с SMS-шлюзом (провайдер определяется отдельно).
- [ ] **Шаг 2 — напоминалка админу.** Каскад напоминаний по заказам, висящим на Шаге 0 дольше порогов (например 3/7/10 дней). Пока без внешних уведомлений клиенту.
- [ ] **Шаг 3 — денежная мотивация админов.** KPI-премия за закрытие заказов из мёртвых статусов в <10 дней. Метрика считается из Шага 0. Вводится только после Шагов 1-2, чтобы у админа были инструменты для дожима.

## Делегирование

- Шаг 0 → Claude Code в `/home/iamsohappy/projects/instrumentburg/apps/calculator`. Оценка: 1-2 дня. Вход: существующие queries в `convex/queue.ts`, `convex/estimates.ts`. Выход: новая страница `/admin/dead-statuses` + Convex query `estimates.listByDeadStatus`.
- Шаг 1 → Claude Code + выбор SMS-шлюза (отдельная info-задача до старта). Оценка: 3-4 дня.
- Шаги 2-3 → после валидации метрики на Шаге 1.
