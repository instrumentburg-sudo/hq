# Интерфейс мастера (очередь + быстрые техзаключения)

## Контекст

Мастера простаивают между задачами — нет явной очерёдности «что чинить следующим», ручной ввод технических заключений медленный. Это фронт Б (простой мастеров) — НЕ связан с застреванием заказов на admin-статусах (см. `calculator-autopilot-stuck-orders`).

Дизайн уже есть: `docs/plans/2026-03-21-master-interface-design.md` (калькулятор) — «фастфуд» UI, быстрые дефектовки, авторотация, голосовой ввод. Память: `memory/project_master_interface.md`.

## Цель

Мастер при включении смены видит «что делать следующим» без вопросов приёмщику. Оформление техзаключения — минимум ручного ввода (типовые формулировки одним кликом). Метрика: сокращение времени между «закрыл заказ N» и «начал заказ N+1», а также среднее время оформления заключения.

## План

- [ ] **Шаг 1 — очередь мастера.** На смене мастер видит список «его» заказов: назначен + следующий в общей очереди. Сортировка по приоритету (дедлайн, возраст заказа, клиент-приоритет). Одна страница без выбора.
- [ ] **Шаг 2 — быстрые карточки техзаключений.** Подстановка типовых формулировок («карбюратор засорён», «не держит прокладка», «сгорела обмотка»), выбор одним кликом, возможность добавить свою. Каталог формулировок пополняется мастерами.
- [ ] **Шаг 3 — авторотация / визуальные триггеры.** Заказ без обновлений >N часов — визуальный сигнал в очереди, опциональное предложение переназначить.

- Этот фронт не конкурирует с автопилотом застрявших заказов за код — они трогают разные части UI (masters vs admins).
